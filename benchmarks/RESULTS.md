## Method

- Per model and scenario: 1 discarded warm-up (its load time is the
  "cold load" column), then 3 measured runs; tables show the **median**.
- TTFT (time to first token) is wall-clock from sending the request to the
  first streamed token, measured client-side on the same host — it includes
  prompt processing but no network latency.
- Token rates come from the `eval_count`/`eval_duration` counters returned
  by the Ollama API. `temperature=0`, `seed=42`; each run's prompt gets a
  unique prefix so Ollama's prompt cache cannot skip prefill.
- VRAM peak is total GPU memory used (sampled every 0.5 s), which includes
  a small desktop baseline. "Placement" is how Ollama split the model
  between GPU and CPU memory.
