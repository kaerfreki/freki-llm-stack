## Using a larger context with OpenCode

Per request (OpenAI-compatible or native), pass `num_ctx`:

```bash
# example: 32k context for a single generate call
curl -s http://localhost:11434/api/generate -d '{
  "model": "ornith:9b",
  "prompt": "...",
  "options": { "num_ctx": 32768 }
}'
```

Persist it on the model so every client (including OpenCode) inherits it:

```bash
cat > /tmp/Modelfile.ornith-32k <<'EOF'
FROM ornith:9b
PARAMETER num_ctx 32768
EOF
ollama create ornith-32k -f /tmp/Modelfile.ornith-32k
# then point OpenCode at model ornith-32k
```

Pick the largest `num_ctx` from the table that still fits your GPU with
headroom for the desktop and any other process. Cold load grows with
`num_ctx` because the KV cache is allocated up front.

## Method

- Per `num_ctx`: unload all models, 1 warm-up (cold load with that
  context), then 3 measured **generation** runs
  (short prompt) and 3 **prefill** runs
  (synthetic dump ≈ 0.75 × num_ctx tokens,
  32 tokens out). Tables show medians.
- TTFT is client-side to the first streamed token; token rates from
  Ollama `eval_*` / `prompt_eval_*` counters. `temperature=0`,
  `seed=42`; unique prompt prefixes defeat the prompt cache.
- VRAM peak is total GPU memory (sampled every 0.5 s), including a
  small desktop baseline.
